# 修改或添加代码的规则（高优先级）

## 1. 注释要求
**所有修改/新增的代码块、函数、方法，必须添加详细注释**：
- 公开函数（大写开头）必须写 GoDoc 风格注释，说明用途、参数、返回值、错误场景
- 非公开函数至少说明用途和关键逻辑
- 复杂逻辑块需要行内注释说明意图

## 2. 测试要求
**所有修改/新增的代码，必须以「函数」为基本单元编写单元测试**：
- 测试文件路径：`test/***_test.go`（与被测代码对应命名）
- 测试函数格式严格为：

```golang
func TestXxx(t *testing.T) {
    // 被测函数或代码逻辑块
}
```

- 未完成测试的代码，不得视为「完成」
