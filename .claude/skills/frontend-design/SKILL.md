本技能指导创建独特的、生产级前端界面，避免通用的"AI 垃圾"美学。实现真实可工作的代码，对美学细节和创意选择投入特殊关注。

用户提供前端需求：一个要构建的组件、页面、应用程序或界面。可能包括关于目的、受众或技术限制的上下文。

## 设计思维

编码前，理解上下文并承诺采用**大胆的**美学方向：
- **目的**：这个界面解决什么问题？谁会使用它？
- **气质**：选择一个极端风格：极简残酷、极简主义混乱、复古未来、有机/自然、奢华/精致、俏皮/玩具般、编辑/杂志风、粗野/原始、装饰艺术/几何、柔和/粉彩、工业/实用等。有很多风格可选。用这些作为灵感，但设计出符合美学方向的作品。
- **约束**：技术要求（框架、性能、无障碍）。
- **差异化**：什么让这个设计**难以忘怀**？别人会记住的一件事是什么？

**关键**：选择清晰的概念方向并精确执行。大胆的极简主义和精致的极简主义都可以奏效——关键是意图，而非强度。

然后实现可工作的代码（HTML/CSS/JS、React、Vue 等），其特点是：
- 生产级且功能完整
- 视觉上引人瞩目且令人难忘
- 与清晰的美学观点相一致
- 在每个细节上精雕细琢

## 前端美学指南

关注以下方面：
- **排版**：选择美观、独特、有趣的字体。避免使用 Arial 和 Inter 等通用字体；选择独特的字体选择来提升前端美学；出人意料、富有个性的字体选择。将独特的展示字体与精致的正文字体配对。
- **色彩与主题**：承诺采用一致的美学。使用 CSS 变量保证一致性。主色加锐利强调色优于胆怯的、均匀分布的调色板。
- **动画**：使用动画实现效果和微交互。优先使用 HTML 的纯 CSS 解决方案。React 中可用时使用 Motion 库。关注高影响时刻：一次精心编排的页面加载和交错显示（animation-delay）比分散的微交互更能愉悦用户。使用滚动触发和出人意料的悬停状态。
- **空间构成**：出人意料的布局。不对称。重叠。对角线流。破网格元素。慷慨的负空间或控制的密度。
- **背景与视觉细节**：创造氛围和深度，而不是默认使用纯色。添加与整体美学相匹配的情境效果和纹理。应用创意形式如渐变网格、噪声纹理、几何图案、分层透明度、戏剧性阴影、装饰边框、自定义光标和颗粒叠加。

**永远不要**使用通用 AI 生成的美学，如过度使用的字体系列（Inter、Roboto、Arial、系统字体）、陈词滥调的配色方案（特别是紫色渐变配白色背景）、可预测的布局和组件模式，以及缺乏特定背景特色的千篇一律设计。

创意解读并做出出人意料的选择，让人感觉这是为背景量身定制的设计。没有两个设计应该相同。在浅色和深色主题、不同字体、不同美学之间变化。**永远不要**在多次生成中收敛到常见选择（例如 Space Grotesk）。

**重要**：将实现复杂性与美学愿景相匹配。极简主义设计需要大量代码、丰富的动画和效果。极简或精致设计需要克制、精确，以及对间距、排版和微妙细节的仔细关注。优雅来自于很好地执行愿景。
