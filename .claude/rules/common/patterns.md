# 通用模式

## 骨架项目

实现新功能时：
1. 搜索经过验证的骨架项目
2. 使用并行 agent 评估选项：
   - 安全评估
   - 可扩展性分析
   - 相关性评分
   - 实现规划
3. 克隆最佳匹配作为基础
4. 在已验证的结构内迭代

## 设计模式

### Repository 模式

将数据访问封装在一致的接口后面：
- 定义标准操作：findAll、findById、create、update、delete
- 具体实现处理存储细节（数据库、API、文件等）
- 业务逻辑依赖抽象接口，而非存储机制
- 支持轻松切换数据源，并使用 mock 简化测试

### API 响应格式

为所有 API 响应使用一致的信封格式：
- 包含成功/状态标识
- 包含数据载荷（错误时为 null）
- 包含错误信息字段（成功时为 null）
- 分页响应包含元数据（total、page、limit）
